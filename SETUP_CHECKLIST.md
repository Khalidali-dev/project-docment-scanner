## ⚠️ Important Notes

### ObjectBox Code Generation
- **Required**: Must run `build_runner build` before first run
- **Automatic**: Includes all 5 data models
- **Models Location**: `lib/models/`
- **Generated Code**: `lib/objectbox.g.dart`

### Android Permissions
The app requires these in `android/app/src/main/AndroidManifest.xml`:
```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
```

### iOS Configuration
Add to `ios/Runner/Info.plist`:
```xml
<key>NSCameraUsageDescription</key>
<string>Camera access is needed to scan documents</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>Photo library access is needed</string>
```

## 🐛 Common Issues & Solutions

### Issue: `objectbox.g.dart` not found
**Solution**: Run `flutter pub run build_runner build`

### Issue: "Build failed" during build_runner
**Solution**: 
```bash
flutter clean
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs
```

### Issue: Camera not working
**Solution**: 
- Check permissions in AndroidManifest.xml
- Request runtime permissions
- Check camera availability

### Issue: ML Kit errors
**Solution**:
- Ensure Google Play Services installed
- Check ML Kit model downloads
- Verify internet connection for first-time model download
